# 化工原理课程设计 — Team 协作实战 / ChemEng Design with AI Team

> CAD Drawing + Team 联动——精馏塔计算→图纸→多轮审查

## 协作模式

```
计算引擎 (dev) ──→ 计算结果
                       ↓
CAD生成 (dev)    ──→ DXF 图纸 ──→ QA 验证
                       ↓               ↓
                  发现问题 ←────── 框外实体/N
                       ↓
                  debugger 定位根因
                       ↓
                  dev 修复 → 再验 → 5轮迭代
                       ↓
                  project-lead 终审 → 交付
```

## 为什么这样设计

| 传统做法 | Team 模式 |
|---------|----------|
| 一个人同时写计算+画图+debug | 三个角色并行，互不阻塞 |
| 图纸画完才发现比例不对 | QA 每轮跑验证脚本 |
| 试错成本：半天 | 试错成本：5分钟 |

## 用到的技能

- **CAD Drawing** — DXF 生成
- **Team** — 四角色调度

## 关键教训

1. 比例是第一决策
2. 验证脚本和生成脚本同步写
3. QA 量化不说模糊话
4. 方案不行快速切换（COM → ezdxf）
